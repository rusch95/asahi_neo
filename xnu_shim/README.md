# xnu_shim — XNU Shim Implementation

## Overview

This directory will contain the XNU shim: a minimal object that links into a custom
XNU kernelcache, intercepts boot control after SPTM initialization, and hands off
to the Linux kernel.

## Status

**Not yet implemented.** SPTM call interface must be reverse-engineered before
any shim code can be written. See docs/SPTM_FINDINGS.md.

## Planned Structure

```
xnu_shim/
├── README.md           (this file)
├── Makefile
├── shim_entry.c        Entry point hooked into XNU startup
├── sptm_iface.h        SPTM call wrappers (generated from RE)
├── sptm_iface.c
├── linux_loader.c      Loads Linux ELF/Image into SPTM-mapped memory
├── fdt_builder.c       Constructs minimal FDT for Linux
└── stub/
    └── kernelcache.xcconfig   Xcode build config for kernelcache linking
```

## Design Constraints

- Must be compiled for ARM64 with `-target arm64-apple-macos` (Mach-O)
- Must link against the kernelcache at the correct base address
- No C library — bare metal / XNU kernel environment only
- Must not call any XNU function that depends on scheduler or VM being active
  (we intercept before those subsystems start)

## Hook Point

Planned intercept at `IOPlatformExpert::start()` (Option A from ARCHITECTURE.md).
This is early enough to control memory layout, late enough that SPTM is fully up.

To hook: replace the kext entry in the kernelcache's `IOCatalogue` with our shim kext.

## Build Requirements (future)

- Xcode command line tools with Apple Silicon target support
- A copy of the XNU kernel headers matching the target macOS/iOS version
- SPTM call interface header (to be generated by scripts/extract_sptm_calls.py)
